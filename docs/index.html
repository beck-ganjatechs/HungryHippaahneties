JWT Security

// JWT with browser fingerprint binding (anti-sidejacking)
const fingerprint = crypto.randomBytes(32).toString('hex');
const fingerprintHash = crypto.createHash('sha256')
  .update(fingerprint).digest('hex');

const token = jwt.sign({
  sub: userId,
  exp: Math.floor(Date.now() / 1000) + 900, // 15 minutes
  fgp: fingerprintHash  // Fingerprint hash in token
}, privateKey, { algorithm: 'RS256' });

// Send fingerprint as hardened cookie
res.cookie('__Secure-Fgp', fingerprint, {
  httpOnly: true,
  secure: true,
  sameSite: 'Strict'
});

Deployment Guide

Deploy HungryHippaahneties to your Kubernetes cluster in minutes.

‚ö†Ô∏è
Prerequisites Required: Kubernetes 1.25+, kubectl, NGINX Ingress Controller, cert-manager, and a CNI with NetworkPolicy support (Calico, Cilium).

Quick Start

terminal 
# Clone repository
git clone https://github.com/example/hungryhippaahneties.git
cd hungryhippaahneties

# Option 1: Deploy using script (recommended)
chmod +x scripts/deploy.sh
./scripts/deploy.sh production

# Option 2: Deploy using kustomize
kubectl apply -k k8s/base/

# Option 3: Deploy using Helm
helm install hungryhippaahneties ./helm/hungryhippaahneties \
  --namespace hungryhippaahneties \
  --create-namespace

# Verify deployment
kubectl get pods -n hungryhippaahneties

Step-by-Step Installation

1. Create Namespace with Pod Security Standards
kubectl apply -f k8s/base/namespace.yaml
kubectl get namespace hungryhippaahneties --show-labels
2. Configure Secrets (External Secrets Manager Recommended)
# Generate secure passwords
DB_PASSWORD=$(openssl rand -base64 32)
REDIS_PASSWORD=$(openssl rand -base64 32)

# Create secrets
kubectl create secret generic db-credentials \
  --namespace hungryhippaahneties \
  --from-literal=POSTGRES_PASSWORD=$DB_PASSWORD
3. Configure TLS Certificates
# Using cert-manager with Let's Encrypt
kubectl apply -f - <
4. Deploy All Resources
kubectl apply -f k8s/base/rbac.yaml
kubectl apply -f k8s/base/network-policies.yaml
kubectl apply -f k8s/base/configmaps.yaml
kubectl apply -f k8s/base/services.yaml
kubectl apply -f k8s/base/deployments.yaml
kubectl apply -f k8s/base/ingress.yaml
5. Verify Deployment
kubectl get pods -n hungryhippaahneties
kubectl get svc -n hungryhippaahneties
kubectl get ingress -n hungryhippaahneties

# Test health endpoint
curl https://hungryhippaahneties.example.com/health
